Briefly — Help & User Guide

Voice ⇄ Text in your browser. Free providers, no account required.

Quick start

  1. Click the Briefly icon in the toolbar to open the side panel (or press ⌘⇧P / Ctrl+Shift+P).
  2. Pick a Speech → Text provider. Web Speech API works with no key.
  3. Hit 🎙 Start recording, speak, then ⏹ Stop — or wait for auto-stop on silence.
  4. For Text → Speech, paste text and pick a voice. StreamElements and Google Translate work with no key.

Settings

  • API keys — paste a key into the matching field. Click Test to verify before saving.
  • Translate button → target language — sets where the 🌐 Translate button sends your last recording. English uses Whisper translation; any other target transcribes then runs a chat-completion translation.
  • Auto-copy transcript — copies the transcript to clipboard the moment transcription finishes.
  • Auto-stop on silence — applies to cloud STT providers, including live streaming.
  • Theme — toggle Dark / Light from the top bar.

Features

  • 🌊 Live waveform + pulsing record button while capturing.
  • ⏹ Auto-stop on silence after 1.5 – 6 s, configurable.
  • 🌐 Translate the last recording to any of 14 languages (Mandarin, Spanish, French, German, Hindi, Japanese, …).
  • 📁 Drag-and-drop audio files to transcribe.
  • ↓ Download recorded audio and synthesized TTS audio.
  • 🕘 History drawer — last 50 entries, click to restore.
  • 🔁 Retry-with-backoff on 429 / 5xx; offline detection.

Troubleshooting

Microphone permission denied

Open chrome://settings/content/microphone and grant access to the extension's origin. Then reload the side panel.

Live streaming doesn't start

Check that the relevant API key is present and tested OK in Settings. Watch the side-panel status text for the exact error (e.g., 403 means an invalid key).

"Audio too short" after recording

Recordings under ~500 bytes are rejected. Speak for at least a second.

Translation fails on non-English target

Non-English targets need a Groq (free) or OpenAI key — the audio is transcribed first, then translated via chat completions.

Privacy

Your API keys live only in chrome.storage.local on this device and are sent only to the provider you select. See the full Privacy Policy for details.
